# Лабораторная работа: Разработка Web3 приложения

Цель работы:  
Продемонстрировать навыки создания децентрализованного приложения (dApp), интегрированного с ранее разработанным Fungible Token смарт-контрактом.  

В ходе работы необходимо:  
 - Применить знания по взаимодействию с блокчейном через Web3-библиотеки (например ethers.js);  
 - Разработать web-приложение для выполнения операций с токеном;  
 - Развернуть приложение на публичном хостинге и предоставить доступ к нему;  

## Общая постановка задачи  
В предыдущей лабораторной работе вы создали и развернули свой собственный FP-токен.  
Теперь необходимо построить вокруг него полноценное dApp, которое позволит пользователям подключать кошелёк (в зависимости от сети, например - MetaMask, Lether, TonKeeper),  
***отображать баланс токенов, позволить выполнять переводы***, а также реализовывать дополнительную логику **согласно выбранному варианту сложности**.

### Вариант 1) Faucet и Burner (Легкий уровень сложности)  ###

Предлагается развенуть два дополнительных контракта, первый для выдачи новым пользователям токенов, второй для "пожертвований".  
Такая схема позволяет любому пользователю запросить небольшое количество токенов (например, 10 токенов) с определённой периодичностью (один раз в сутки).  
Если ваш токен реализует функцию сжигания (burn), то пользователь может сжечь часть своих токенов напрямую, если нет, то через пожертвования на специальный новый контракт "burner".  

**В качестве дополнительной логики необходимо показывать на UI'e общее количество сожжённых токенов.**

### Вариант 2) Простое DAO для голосования (Средний уровень сложности) ###

Необходимо создать контракт организации "DAO", который позволяет создавать предложения (proposals) и голосовать за них с помощью токенов.  
Владельцы токенов могут отдать свой голос «за» или «против» (вес голоса равен количеству токенов на момент голосования).  
По окончании голосования можно выполнить действие (например, перевести средства из казны или изменить какой-нибудь параметр).  
Естественно потребуется развернуть дополнительный контракт голосования, связанный с токеном, можно взять за основу контракт [Governor из OpenZeppelin](https://docs.openzeppelin.com/contracts/5.x/governance), связав его с токеном (который должен поддерживать стандарт `ERC20Votes`).  
Для упрощения можно сделать простой контракт голосования с фиксированным набором предложений, где голоса считаются на основе баланса токенов. Контракт должен хранить структуру `Proposal` (описание, варианты голосования, deadline, результаты) и функции `createProposal`, `vote`, `executeProposal`.  
Для проверки голосов контракт может обращаеться к изначально развёрнутому токену и проверять баланс пользователя.

**В качестве дополнительной логики необходимо показывать на UI'e все Proposal'ы DAO и дать возможность пользователям участвовать в них.**

Полезные ссылки по этому варианту:  
https://github.com/0xOptimism/Awesome-DAO

### Вариант 3) Простой пул ликвидности (аналог Uniswap V2) (Сложно (!))

Необходимо предоставить возможноть создавать пулы для пар «Токен — валюта сети»  
Пользователи могут добавлять ликвидность, получая взамен токены пула (LP tokens).  
**В качестве дополнительной логики реализуется функция обмена (swap): пользователь отправляет валюту сети и получает токены, или наоборот.**

Для упрощения можно сделать контракт с функциями `addLiquidity` (принимает валюту сети или токены, выдаёт LP-токены) и `swapTokens` / `swapNativeAsset`.  
LP-токены можно реализовать как отдельный Fungible Token, выпускаемый при добавлении ликвидности.  
**Это довольно объёмная задача**, поэтому допустимо использовать упрощённую логику (без учёта комиссий и сложных расчётов)...

## Развертывание приложения на хостинге ##

Для размещения статического сайта можно использовать GitHub Pages или Render.com (как на дисциплине веба ;)).  
т.к. вся backend-логика будет реалзиована на стороне контрактов - необходимость в хостинге ограниченна предложенными решениями.  
В качестве узла для обработки запросов рекомендуется использовать те ноды, которые поддерживаются выбранными вами кошельками.
