# Лабораторная работа: Межсетевой мост/оракул для кросс-чейн взаимодействия #

## Цель работы: ##
Продемонстрировать практическую реализацию межсетевого взаимодействия **гетерогенных** блокчейнов.  
Лабораторная работа предполагает кросс командное взаимодействие.  
Создать децентрализованный мост с использованием оракула, который позволяет пользователю обменять токен в одной сети на эквивалентный токен в другой сети и затем использовать его в ваших приложениях (для голосования в DAO, для получения ликвидности ну либо просто сжечь...)

## Задачи: ##
1. Разработать смарт-контракты для:
 - Токена в исходной сети (например, VoteTokenETH)
 - Обёрнутого токена в целевой сети (WrappedVoteTokenXLM)
 - Прокси-контракта DAO с функцией голосования в обеих сетях

2. Разработать оффчейн-компонент (Oracle + Bridge):
 - Слушатель событий в сети Ethereum
 - Валидатор транзакций (минимально: достаточно подождать пока транзакция включится в блок) 
 - REST API Client для эмиссии токенов в соседней сети.
 - (опционально) обратный оракул для синхронизации результатов голосования (в случае варианта с DAO)

3. Реализовать механизм атомарного обмена через оракул без использования центрального кастодиана (схема lock–mint либо mint-burn если стандарты совпадают).

4. Протестировать полный цикл:

- Пользователь блокирует свои токены → получает другие обёрнутые токены.
- Голосует через прокси-контракт / Добавляет ликвидность / делает burn
- (Для варианта с DAO) Результат голосования передаётся обратно в исходную сеть.


## Технические рекомендации ##
Для реализации схемы lock-mint вам понадобится добавить в ваш контракт Fungible Token'a функцию lock(uint256 amount) – которая Emit'ит событие Locked, которое и будет служить триггером для запуска механизма общения ваших off-chain компонентов.

## Состав отчёта (сдаваемые материалы) ##
Исходный код:
 - Смарт-контракты сети А
 - Смарт-контракты сети Б
Оффчейн-мост:
 - Прослушиватель эвентов в сети / оракул
 - Валидатор транзакций
 - Минтилка.
 - Docker-compose (опционально)
 - Не секьюрные переменные среды окружения (Куда подключаетесь - ChainID, RPC URL, адреса контрактов)

### Демонстрация работоспособности: ###

По одному человеку от каждой команды демонстрируют интеграцию по одному из направлений.
Студент из команды А демонстрирует способ получения и возможность использования wrapped токена команды Б в своей сети,  
А студент из команда Б демонстрирует то же самое только с использованием wrapped токена команды А в своей сети.

В идеале если вы приложите скриншоты или видео с шагами, где видно:
 - Логи оракула (получение событий, валидация, вызовы mint).
 - Баланс токенов пользователя до/после в обоих блокчейнах (через блокчейн-эксплореры).
 - Транзакция голосования и проверка учёта голоса.
 - Хеши всех ключевых транзакций (lock, mint, vote).
