# Лабораторная работа: Разработка и развертывание ERC20 токена в тестовой сети

Цель работы:  
Получить навыки разработки и развёртывания смарт-контрактов на платформах распределённого реестра  

## Создание кошелька и получение тестовых средств
Для начала работы вам необходимо установить кошелёк (удобный вариант будет расширение для браузера).  
Создайте новый кошелек (или используйте существующий, предназначенный только для тестов).  
Переключите сеть на нужную тестовую сеть (например, Sepolia, если ваш вариант - сеть Ethereum). 
Добавьте сеть вручную, если её нет в списке, используя данные с https://chainlist.org/  

Найдите один из публичных кранов для выбранной сети, например:  
Sepolia (Ethereum): https://faucet.quicknode.com/ethereum/sepolia  
Mumbai (Polygon): https://mumbaifaucet.com/  

Введите адрес своего кошелька и запросите средства.

При использовании публичных faucet могут быть лимиты по времени и IP, поэтому для накопления нужного объёма для оплаты газа/комиссии попросите ваших коллег по группе помочь :)

## Подготовка среды разработки
В зависимости от вашей сети вы можете выбрать любой доступный вариант, но я бы рекомендовал пользоваться инструментами разработки общего назначения. Для EVM-совместимых сетей можно рассмотреть вариант среды разработки - Hardhat.  
Экспортируйте ваш приватный ключ от кошелька для того чтобы среда разработки имела возможность разворачивания контракта в сети от вашего лица.  
**В целях безопасности не храните приватные ключи в коде. Используйте переменные окружения.**  
**Не публикуйте файл .env в репозиторий. Добавьте его в .gitignore.**  

## Написание смарт-контракта
В рамках лабораторной работы необходимо разработать токен по стандарту ERC-20 на основе спецификации предложенной [OpenZeppelin](https://docs.openzeppelin.com/contracts/5.x/erc20)

Детали реализации:
- Должна быть возможность присвоить имя токена и символ.
- Контракт должен хранить адрес владельца как того кто развернул его.
- Владелец контракта должен иметь возможность производить эмиссию (т.е. контракт должен быть *mint***able**)
- опционально - позволять держателям сжигать токены.

## Развертывание контракта в тестовой сети

Произведите развёртывание контрактра с помощью среды разработки либо вручную.  
Удостовертесь что транзакция выполнилась успешно и перейдите на адрес контракта в обозревателе блокчейна тестой сети (EtherScan и подобные).  
Чтобы код контракта был виден в обозревателе, добавьте ABI и выполните верификацию согласно инструкциям самого обозревателя, либо автоматизированно, с помощью инструментов разработчика по типу HardHat'a (понадобится добавление API Key'я от обозревателя).

## Эмиссия токенов

**Условие для командной работы: студенты должны обменяться адресами и продемонстрировать работоспособность переводов.**

Теперь, когда контракт развёрнут, вы можете выпустить токены и отправить их своим коллегам. Взаимодействовать с контрактом можно не только через консоль, но и через Remix IDE, подключив его к кошельку в браузере (например MetaMask).

## Требования к отчету о выполненной лабораторной работе

- Выбранный ChainId в качестве тестовой
- Краткое описание выполненных шагов
- Ссылку на ваш верифицированный контракт в обозревателе сети
